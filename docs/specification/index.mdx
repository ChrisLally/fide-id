Fide IDs give every entity a stable, protocol-level identity. You can think of them as three-part addresses: entity type, reference type, and a fingerprint of the [reference](#reference).

<Callout title="Entity-Specific Rules">
  See [Fide Vocabulary](/vocabulary) for per-entity-type Fide ID rules.
</Callout>

## Fide ID Format [#fide-id-format]

**Shape:** `did:fide:0x{entityType}{referenceType}{referenceFingerprint}`

- MUST be exactly 51 characters long.

### Prefix [#prefix]
Declares that the identifier is a DID using the Fide method namespace.

- MUST start with `did:fide:0x` in compliance with [DID](https://www.w3.org/TR/did-core/).

### Entity Type [#entity-type]

Defines what kind of entity is being identified.

- MUST use a valid 2-character hexadecimal [FCP Entity Type Code](/vocabulary).

### Reference Type [#reference-type]

Defines what kind of [reference](#reference) the identifier is derived from.

- MUST use a valid 2-character hexadecimal [FCP Reference Type Code](/vocabulary).

### Reference Fingerprint [#reference-fingerprint]

Defines the fingerprint derived from the [reference](#reference).

- MUST derive by [UTF-8](https://www.rfc-editor.org/rfc/rfc3629) encoding the [reference](#reference), hashing it with [SHA-256](https://csrc.nist.gov/publications/detail/fips/180/4/final), and using the first 36 hexadecimal characters of the hash to produce the reference fingerprint.

### Reference [#reference]

Defines the reference identifier.

- MUST be a string identifier.
- MUST match the declared [Reference Type](#reference-type).
- SHOULD uniquely identify the declared [Entity Type](#entity-type) when interpreted with the declared [Entity Type](#entity-type) and [Reference Type](#reference-type) together.
